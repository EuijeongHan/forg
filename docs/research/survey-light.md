# 설문 B — 라이트 버전 (2~3분, 콜드 배포용)

> 용도: 커뮤니티·단체방 등 **직접 아는 사이가 아닌 대상**에게 뿌릴 때.
> 원칙: 필수 7문항 + 선택 1문항, 서술 0개, 사례 섹션 없음(민감도·이탈 최소화).
> 도입부·개인정보 동의 문구는 풀 버전(survey-full.md)과 동일하게 사용.

---

**Q1.** 현재 역할은? *(필수, 단일)*
- 애널리스트(셀) / 애널리스트·운용역(바이) / 펀드매니저 / RA·리서치 지원 / 기타: ___

**Q2.** 지속 모니터링하는 상장사 수는? *(필수, 단일)*
- 10개 미만 / 10~20개 / 21~40개 / 41개 이상

**Q3.** 공시를 접하는 **주된** 경로 하나는? *(필수, 단일)*
- DART 직접 / 증권사 HTS·MTS 알림 / 단말(Bloomberg 등) / 사내 시스템 / 텔레그램 채널 / 뉴스 / 기타: ___

**Q4.** 최근 1년 내 중요 공시를 놓치거나 늦게 발견한 적이 있습니까? *(필수, 단일)*
- 여러 번 있다 / 한두 번 있다 / 없다 / 답하기 어렵다

**Q5.** 현재 방식의 가장 큰 불만 하나는? *(필수, 단일)*
- 알림 과다로 중요한 게 묻힘 / 발견이 늦음 / 내용 파악이 오래 걸림 / 정정공시 변경점 찾기 어려움 / 팀 확인 여부를 알 수 없음 / 불만 없음 / 기타: ___

**Q6.** 다음 중 **딱 하나만** 쓸 수 있다면? *(필수, 단일)*
- 중요 공시 강조 + 미확인 재알림
- 핵심 숫자 카드 + 원문 링크
- 정정 전후 자동 비교
- 이벤트 타임라인·향후 D-day
- 팀 공유·담당자 확인
- 장 마감 일간 리캡

**Q7.** 회사 정책상 외부 메신저 봇에 관심기업 목록 등록이 가능한가요? *(필수, 단일)*
- 가능할 듯 / 조심스럽다 / 불가능할 듯 / 모름

**Q8.** 테스트 버전을 써보시겠어요? *(필수, 단일)*
- 네 → 개인정보 동의(풀 버전 섹션 9와 동일) → Telegram 아이디(선택)
- 완성되면 보고 싶다 → 동의 → 아이디(선택)
- 아니요 → 종료

## 메모
- Q6은 풀 버전 Q16·Q17을 하나로 압축한 강제 단일 선택 — 라이트에선 이 신호 하나만 취한다.
- 라이트 버전 응답자 중 Q8 '네'는 풀 버전 링크 또는 인터뷰로 후속 연결.
