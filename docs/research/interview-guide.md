# 설문 C — 심층 인터뷰 가이드 (30~40분, 반구조화)

> 용도: 동생(Geonsoo) 및 테스트 참여 의향자 1:1. **설문으로는 결정하면 안 되는 것들**을
> 여기서 푼다: 팀 확인(ack)의 위치, 컴플라이언스 통과 조건, 지불 의사, 워크플로 통합.
> 원칙: 유도하지 않기 — 기능 설명 전에 현재 행동부터. "~하면 쓰실래요?"는 마지막에만.

---

## 0. 오프닝 (2분)
- 목적 고지: "도구를 팔려는 게 아니라, 공시 확인 업무가 실제로 어떻게 돌아가는지 배우려는 것."
- 녹음/메모 동의. 회사·고객 특정 정보는 말하지 않아도 됨을 안내.

## 1. 하루 워크플로 재구성 (8분) — 가장 중요
- "어제(또는 최근 평일) 아침부터, 공시를 언제·어떤 도구로 확인하셨는지 순서대로 말씀해 주세요."
- 후속: 장 시작 전 루틴? 장중 알림은 어디로 오나? 장 마감 후엔?
- "그 과정에서 가장 시간을 잡아먹는 건 뭔가요?"
- 도구별로: DART 알림 설정을 실제로 쓰는지, 몇 개 기업까지, 왜 그 도구인지.

## 2. 누락 사례 딥다이브 (8분)
- (설문 응답이 있으면 이어서) "그 사례를 처음부터 끝까지 들려주세요 — 공시가 나온 시점부터 알게 된 순간까지."
- 후속: 그날 알림은 몇 개나 왔나? 왜 하필 그게 묻혔나? 팀에선 어떻게 됐나(질책·프로세스 변경)?
- **반사실 질문**: "무엇이 있었다면 그날 잡아냈을까요?" (선택지 제시하지 말고 먼저 듣기)

## 3. 팀 구조와 확인 책임 (7분) — Stage 4 스코프 결정 입력
- "공시 확인은 팀에서 누구 책임인가요? 지정 담당·순번·각자?"
- "중요 공시를 보면 팀에 어떻게 공유하나요? (사내 메신저? 텔레그램 방? 구두?)"
- "'누가 확인했는지'가 보이면 좋겠다는 생각, 실제로 해보신 적 있나요? 그게 **어디에** 떠야 쓸모 있나요 — 텔레그램 방? 사내 메신저? 별도 화면?"
- 재알림: "안 읽었을 때 다시 울리는 건 도움인가요, 소음인가요? 몇 번까지 허용?"

## 4. 컴플라이언스 (5분) — 배포 모델 결정 입력
- "외부 봇에 관심기업 목록을 넣는 것, 회사에서 문제 될까요? 실제 규정을 아시나요, 감인가요?"
- "문제가 된다면 어떤 형태면 통과 가능한가요? (개인 기기? 종목명 대신 코드? 사내 설치형?)"
- "과거에 외부 도구 도입이 막힌 사례가 있나요? 왜였나요?"

## 5. 기능 반응 (5분) — 여기서 처음 기능 언급
- 카드 3장을 보여주고(중요도 강조+재알림 / 숫자 카드+정정 비교 / 타임라인+D-day): "본인 업무 기준으로 순서를 매긴다면? 왜?"
- "이 중 '없어도 그만'인 건 뭔가요?" (탈락 신호가 채택 신호보다 정직하다)

## 6. 가치·지불 (3분) — 직접 가격 묻지 않기
- "이 문제를 지금 해결하기 위해 쓰는 게 있나요? (유료 단말·서비스·사람의 시간)"
- "회사 경비로 도구를 결제하려면 어떤 절차를 거치나요? 개인 결제로 쓰는 도구도 있나요?"

## 7. 클로징 (2분)
- "제가 안 물어봤지만 말해주고 싶은 것?"
- 테스트 참여 안내 + 다음 인터뷰 소개 요청(스노우볼).
